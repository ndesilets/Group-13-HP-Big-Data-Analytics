Indexes and Partitioning

This set of experiments was designed to quantify the effect of indexing and partitioning strategeys on bulk DML operations. An index is a schema object that contains an entry for each value that appears in the indexed column(s) of a table and provides direct, fast access to rows upon query. However, maintaining an index is extremely costly when performing bulk inserts on a million or billion row table. In order to investigate this effect we looks at global and local indexing, considering both the prefix and non-prefix index types. If a table is partitioned by date, then a prefixed index would have the leftmost column of date.

In order to create experiments to test indexing we created partitioned and subpartitioned tables that would allow us to see if there was any benefit in prefix vs. non-prefixed index. We also simulated our clients production environment by creating a data reader table, roughly 200 million rows, and a data loader table that was 5% of that (10 million).
